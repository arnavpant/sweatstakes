## Spacing Scale

Unchanged from Phase 1. 8-point scale, all values multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge spacing |
| sm | 8px | List item internal gap (avatar-to-text), compact element spacing |
| md | 16px | Default horizontal page padding, card internal padding |
| lg | 24px | Section vertical gap, card padding on leaderboard rows |
| xl | 32px | Between major page sections (streak → leaderboard → rewards) |
| 2xl | 48px | Page top padding (below fixed bottom nav safe area) |
| 3xl | 64px | Bottom padding to clear fixed bottom nav |

Exceptions (inherited + new):
- Bottom nav touch targets: minimum 44px height (`min-h-[44px]`) — inherited from Phase 1
- Safe area insets: `padding-bottom: env(safe-area-inset-bottom)` on bottom nav wrapper — inherited
- "+" add reward FAB / button: minimum 44x44px touch target
- Leaderboard row height: minimum 56px (`min-h-14`) to meet touch target for future tap interaction

## Typography

Unchanged from Phase 1. Single typeface, exactly 2 weights. Maximum 4 size roles — Phase 4 uses exactly 4.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 40px (text-5xl) | 700 (bold) | 1.1 | Not used in Phase 4 |
| Heading | 24px (text-2xl) | 700 (bold) | 1.2 | Page title "Streaks & Balance", section headings ("Your Streak", "Balances", "Rewards") |
| Body | 16px (text-base) | 400 (regular) | 1.5 | Member names, reward item names, empty state body, dialog body copy including current balance line |
| Label | 12px (text-xs) | 700 (bold) | 1.4 | Point balance values, "pts" suffix labels, streak count sub-labels, nav labels, personal best value, field labels |

Point balance display exception: balance numbers use 20px (text-xl) weight 700 for visual emphasis in the leaderboard — this is a named exception, not a new role. Section headings "Balances" and "Rewards" also use 20px / bold (text-xl) as a sub-heading within the Heading role range.

**14px is not used anywhere in Phase 4.** All elements previously specified at 14px are mapped to either 12px (Label) or 16px (Body) as documented below.

Font loading: `weights: [400, 700]` — no additional weights.

## Color

Royale token system from `globals.css` — unchanged from Phase 1.

### 60/30/10 Split

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#001233` (`background` / `surface`) | Page background, scrollable content area |
| Secondary (30%) | `#0c1f40` (`surface-container`) / `#18294b` (`surface-container-high`) | Section cards, leaderboard row backgrounds, bottom sheet background, dialog background |
| Accent (10%) | `#66dd8b` (`secondary`) | See accent reserved-for list below |
| Destructive | `#ffb4ab` (`error`) on `#93000a` (`error-container`) | Delete reward action only |

**Accent reserved for (explicit list — not general interactive elements):**
1. Streak count text when streak >= 4 weeks (`text-secondary`) — inherited from `streak-counter.tsx`
2. Day dots when checked in (`bg-secondary`) — inherited from `day-dots.tsx`
3. Progress count text when goal met (`text-secondary`) — inherited from `day-dots.tsx`
4. "Add Reward" "+" button background (`bg-secondary`, `text-on-secondary`)
5. "Confirm Redeem" button background in dialog (`bg-secondary`, `text-on-secondary`)
6. Active bottom nav icon — inherited from Phase 1
7. "Personal best" label badge background (`bg-secondary/20 text-secondary`) — new in Phase 4
8. Positive balance display: balances > 0 render in `text-secondary`
9. Current user's leaderboard row ring highlight (`ring-1 ring-secondary/30`) — focal point for Streaks & Balance page

**Accent is NOT applied to:** negative balances, disabled redeem button, delete buttons, leaderboard row borders, inactive list items, dismiss buttons.

**Negative balance color:** `text-error` (#ffb4ab) — negative balances display in error color to signal debt. This is a semantic color use, not destructive-action use.

**Disabled state:** `opacity-50` on Redeem button when `balance < cost` — no color change, just opacity.

## Screen Specifications

### Streaks & Balance Page (`/streaks`)

**Layout:** Single scrollable column. `px-4 pt-6 pb-32 space-y-8`. The `pb-32` clears the fixed bottom nav.

**Page title:** "Streaks & Balance" — Heading size (24px / bold), `text-on-surface`, at top of scroll content before section 1.

**Focal point:** The current user's leaderboard row is the primary visual anchor of this page. It is distinguished from other rows by a `ring-1 ring-secondary/30` highlight and its balance value rendered in `text-secondary` (if positive). When the user opens this page, their eye should land on their own position and balance first.

**Page structure (top to bottom):**

1. **Streak Section** — reuses `StreakCounter` component pattern
2. **Member Balance Leaderboard** — new section
3. **Rewards Menu** — new section

---

#### Section 1: Streak

**Container:** `bg-surface-container rounded-xl p-4 space-y-3`

**Current streak row:** Reuse existing `StreakCounter` component from `src/components/dashboard/streak-counter.tsx`. Pass `streak` prop as before. Displays: `🔥 N weeks streak` or `No streak yet`.

**Personal best row:** Below the current streak row.
- Label: "Personal best" — 12px / bold, `text-on-surface-variant`
- Value: `🏆 N weeks` — **12px / bold** (`text-xs font-bold`), `text-on-surface`
- If personal best is 0: show "No best yet" — `text-on-surface-variant`
- If current streak equals personal best AND is > 0: append badge `bg-secondary/20 text-secondary text-xs px-2 py-0.5 rounded-full` with text "Current best!"

---

#### Section 2: Member Balance Leaderboard

**Section heading:** "Balances" — 20px (text-xl) / bold, `text-on-surface`

**Section header row:** heading on left + "(updated after weekly settlement)" — 12px / regular, `text-on-surface-variant` on right, vertically centered.

**Leaderboard list:** `space-y-2` between rows.

**Each leaderboard row:** `bg-surface-container rounded-xl px-4 py-3 flex items-center gap-3 min-h-14`

Row contents left to right:
1. **Rank number:** `text-sm font-bold text-on-surface-variant w-5` — "1", "2", "3", etc.
2. **Avatar:** `w-9 h-9 rounded-full` — Google profile picture via `<img>` with `alt="{displayName}"`. Fallback: single initial in `bg-surface-container-high rounded-full flex items-center justify-center text-sm font-bold text-on-surface-variant`.
3. **Display name:** `text-base font-regular text-on-surface flex-1` — truncated with `truncate` if too long. Append "(You)" in `text-on-surface-variant` if this row is the current user.
4. **Balance:** right-aligned. Format: `text-xl font-bold` — color `text-secondary` if > 0, `text-error` if < 0, `text-on-surface-variant` if exactly 0. Followed by `text-xs font-bold text-on-surface-variant` " pts" suffix on the same line.

**Sorting:** Rendered server-side, sorted descending by balance. First place = highest balance.

**You indicator:** The current user's row gets `ring-1 ring-secondary/30` on the container to subtly highlight their position. This is the page's primary focal point (see Focal point above).

---

#### Section 3: Rewards Menu

**Section heading:** "Rewards" — 20px (text-xl) / bold, `text-on-surface`

**Section header row:** heading on left + "+" button on right.

**"+" Add Reward button affordance:** The "+" icon button sits inline with the "Rewards" section heading, positioned flush right. The section heading provides the label context — the "+" communicates "add to this section." No separate visible text label is needed because the heading immediately to the left reads "Rewards," making the compound affordance unambiguous: "+ [to] Rewards." The button carries `aria-label="Add reward"` for screen readers.

**"+" Add Reward button:** `bg-secondary text-on-secondary rounded-full w-8 h-8 flex items-center justify-center font-bold text-lg min-w-[44px] min-h-[44px]` — tapping opens the Add Reward bottom sheet. Icon: `Plus` from lucide-react, size 18px. `aria-label="Add reward"`.

**Empty state (no rewards yet):**
- `bg-surface-container rounded-xl p-6 text-center space-y-2`
- Icon: `lucide-react` `Gift` at 32px, `text-on-surface-variant`
- Heading: "No rewards yet" — 16px / bold, `text-on-surface`
- Body: "Add something worth working for." — 16px / regular, `text-on-surface-variant`

**Reward item list:** `space-y-2`

**Each reward row:** `bg-surface-container rounded-xl px-4 py-3 flex items-center gap-3 min-h-14`

Row contents left to right:
1. **Reward name:** `text-base text-on-surface flex-1 truncate`
2. **Cost badge:** `bg-surface-container-high text-on-surface text-sm font-bold px-2 py-1 rounded-full` — "{N} pts"
3. **Redeem button:** `text-sm font-bold rounded-full px-3 py-1.5 min-h-[36px]`
   - If `userBalance >= cost`: `bg-secondary text-on-secondary` — label "Redeem"
   - If `userBalance < cost`: `bg-surface-container-high text-on-surface-variant opacity-50 cursor-not-allowed` — label "Redeem" (disabled, no click handler)
4. **Delete button:** `text-error p-1.5 rounded-full hover:bg-error/10 min-w-[36px] min-h-[36px] flex items-center justify-center` — icon: `lucide-react` `Trash2` at 16px. Visible to all members (D-15 — trust-based, no owner role).

**Insufficient balance hint:** When tapping a disabled Redeem button: show no toast — the button is visually disabled. The "You need N more points" message (D-20) displays only inside the Redeem Dialog when opened via a programmatic path (not applicable when button is disabled). Disabled button is sufficient affordance.

### Add Reward Bottom Sheet

**Component:** Base UI `Drawer` from `@base-ui/react` — slides up from bottom.

**Trigger:** "+" button in Rewards section header.

**Backdrop:** `bg-black/50 fixed inset-0` — standard modal backdrop.

**Sheet container:** `fixed bottom-0 left-0 right-0 bg-surface-container-high rounded-t-2xl p-6 space-y-4 pb-safe` (bottom safe area inset). Max width handled by responsive container.

**Sheet handle:** `w-10 h-1 bg-outline-variant rounded-full mx-auto mb-4` — standard bottom sheet handle visual.

**Sheet title:** "Add a Reward" — 20px / bold, `text-on-surface`

**Form fields (react-hook-form + zod):**

1. **Reward Name field:**
   - Label: "Reward name" — 12px / bold, `text-on-surface-variant uppercase tracking-widest`
   - Input: full-width, `bg-surface-container rounded-xl px-4 py-3 text-base text-on-surface placeholder:text-on-surface-variant border border-outline-variant focus:border-secondary focus:ring-2 focus:ring-secondary/30 outline-none`
   - Placeholder: "e.g. Movie night pick"
   - Max length: 60 characters (enforced in zod schema)

2. **Point Cost field:**
   - Label: "Point cost" — 12px / bold, `text-on-surface-variant uppercase tracking-widest`
   - Input: same styling as above, `type="number"` with `inputMode="numeric"` for mobile keyboard
   - Placeholder: "e.g. 3"
   - Min: 1. Zod: `z.number().min(1)`
   - Suffix hint: "pts" displayed inline right of input via `relative` container

**Validation errors:** Inline below each field. `text-error text-sm mt-1`. Renders only after first submit attempt (react-hook-form `mode: "onSubmit"`).

**Action buttons (row, right-aligned):**
1. Discard — `variant="ghost"` Button, label "Discard" — dismisses sheet, unsaved reward data is discarded
2. Add — `bg-secondary text-on-secondary font-bold rounded-full px-6 py-2` — submits form. Loading state: spinner (`Loader2` lucide, `animate-spin`) replaces label, button disabled.

---

### Redemption Confirmation Dialog

**Component:** Base UI `Dialog` from `@base-ui/react`.

**Trigger:** Tapping an enabled "Redeem" button on a reward row.

**Backdrop:** `bg-black/60 fixed inset-0` — slightly darker than bottom sheet backdrop to distinguish layering.

**Dialog container:** `bg-surface-container-high rounded-2xl p-6 space-y-4 mx-4 max-w-sm w-full` — centered on screen.

**Dialog title:** "Redeem Reward?" — 20px / bold, `text-on-surface`

**Dialog body:**
- Line 1: `"[Reward Name]"` — 16px / bold, `text-on-surface` — quoted reward name
- Line 2: "This will cost you [N] pts from your balance." — 16px / regular, `text-on-surface-variant`
- Line 3 (current balance): "Your current balance: [X] pts" — **12px / bold** (`text-xs font-bold`), `text-on-surface-variant` — rendered as a supporting label beneath the main body copy

**Action buttons (row, full-width, stacked vertically on mobile):**
1. Confirm Redeem — `bg-secondary text-on-secondary font-bold rounded-full w-full py-3` — label "Confirm Redeem". Loading state: spinner replaces label, button disabled.
2. Keep my points — `variant="ghost"` Button, full-width, label "Keep my points" — dismisses dialog, no action taken, user retains their current balance.

**Post-redemption:** Dialog closes immediately after server action resolves. No success toast — balance update visible on the leaderboard is the feedback (D-21: silent redemption).

### Settings Page — Timezone / Settlement Hour Addition

**Placement:** New section appended below the existing goal stepper section. Section heading: "Settlement Settings" — 20px / bold, `text-on-surface`.

**Section container:** `bg-surface-container rounded-xl p-4 space-y-4`

**Timezone picker:**
- Label: "Group timezone" — 12px / bold, `text-on-surface-variant uppercase tracking-widest`
- Control: Native `<select>` element populated from `Intl.supportedValuesOf('timeZone')`, styled with Tailwind tokens: `bg-surface-container-high text-on-surface rounded-xl px-4 py-3 w-full border border-outline-variant focus:border-secondary focus:ring-2 focus:ring-secondary/30 outline-none text-base`
- Current value shown: IANA timezone string, e.g. "America/New_York"
- Default: "America/New_York"
- Note below: "Changes affect the whole group." — 12px / regular, `text-on-surface-variant`

**Settlement hour picker:**
- Label: "Settlement time (Monday)" — 12px / bold, `text-on-surface-variant uppercase tracking-widest`
- Control: Native `<select>` with 24 hourly options (00:00 → 23:00) in the selected timezone, styled identically to timezone picker
- Default: "05:00" (5:00 AM)
- Note below: "The week settles Monday at this time in your group's timezone." — 12px / regular, `text-on-surface-variant`

**Save button:** `bg-secondary text-on-secondary font-bold rounded-full w-full py-3` — label "Save settlement settings". Loading state: spinner. Disabled when no changes made (same values as saved state).

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Touch targets | Minimum 44x44px on all interactive elements |
| Leaderboard rows | `min-h-14` (56px) — not interactive in Phase 4; future-proofed |
| Add reward button | `aria-label="Add reward"` — icon-only button requires label |
| Delete button | `aria-label="Delete [reward name]"` — icon-only button requires label |
| Redeem button (disabled) | `aria-disabled="true"` + visual opacity — do NOT use `disabled` attribute on Base UI button to preserve focus |
| Dialog accessibility | Base UI Dialog handles `role="dialog"`, `aria-modal="true"`, focus trap, Escape to close |
| Bottom sheet accessibility | Base UI Drawer handles `role="dialog"`, focus trap, swipe-to-close gesture |
| Balance sign | Negative balances: prefix with "−" (minus sign U+2212) not hyphen for screen readers |
| Color contrast | `on-surface` (#d8e2ff) on `surface-container` (#0c1f40) — 8.5:1 (WCAG AAA) |
| Color contrast (accent on dark) | `on-secondary` (#003919) on `secondary` (#66dd8b) — 8.1:1 (WCAG AAA) |
| Color contrast (error) | `error` (#ffb4ab) on `background` (#001233) — 5.2:1 (WCAG AA) |
| Form labels | Explicit `<label htmlFor>` associations on all form inputs |
| Loading states | `aria-busy="true"` on submitting buttons |
