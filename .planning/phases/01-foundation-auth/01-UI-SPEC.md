## Spacing Scale

8-point scale. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge spacing |
| sm | 8px | Button internal gap (icon + label), compact list item padding |
| md | 16px | Default horizontal page padding, form field internal padding |
| lg | 24px | Card internal padding, section vertical gap |
| xl | 32px | Between major sections on login screen |
| 2xl | 48px | Top/bottom breathing room on login screen (above logo, below button) |
| 3xl | 64px | Page-level vertical centering offset |

Exceptions:
- Bottom nav touch targets: minimum 44px height (iOS HIG minimum tap target) — use `py-3` (12px) on nav buttons with `min-h-[44px]` guard
- Safe area insets: `padding-bottom: env(safe-area-inset-bottom)` added to bottom nav wrapper — not a spacing token, a CSS env value (CONTEXT.md D-21)
- PWA standalone mode: `padding-top: env(safe-area-inset-top)` on root layout for Dynamic Island / notch clearance (CONTEXT.md D-21)

## Typography

Single typeface: Plus Jakarta Sans throughout.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 40px (text-5xl) | 800 (extrabold) | 1.1 | Login screen headline ("Ready to sweat?") |
| Heading | 24px (text-2xl) | 700 (bold) | 1.2 | Dashboard greeting ("Welcome, Arnav!"), section titles |
| Body | 16px (text-base) | 400 (regular) | 1.5 | Body copy, placeholder text, error messages |
| Label | 12px (text-xs) | 600 (semibold) | 1.4 | Form labels (uppercase + tracking-widest), nav labels, badges |

Weight contract: exactly 2 weights in use — regular (400) and bold/extrabold (700/800 treated as one "heavy" tier). Semibold (600) is permitted on labels only as a special case for legibility at 12px.

## Color

Full Royale token system from stitch-copy-paste-code-inspiration.txt, wired into Tailwind v4 `@theme` in `globals.css`.

### 60/30/10 Split

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#001233` (`background` / `surface`) | Page backgrounds, main content area |
| Secondary (30%) | `#0c1f40` (`surface-container`) / `#071a3b` (`surface-container-low`) / `#18294b` (`surface-container-high`) | Cards, glass panels, bottom nav background |
| Accent (10%) | `#66dd8b` (`secondary` / `tertiary`) | See accent reserved-for list below |
| Destructive | `#ffb4ab` (`error`) on `#93000a` (`error-container`) | Destructive actions only (sign-out confirmation if ever added) |

**Accent reserved for (explicit list — not general interactive elements):**
1. Google sign-in button background gradient (`from-secondary to-on-tertiary-container`)
2. Active bottom nav icon (filled icon + emerald-glow shadow)
3. Emerald glow decorative effect on the logo icon container
4. Focus ring on form inputs (`focus:ring-secondary/50`)
5. Hyperlinks and secondary text CTAs (e.g., error retry links)

**Accent is NOT applied to:** inactive nav icons, card borders, body text, disabled states.

## Screen Specifications

### Login Screen

**Layout:** Full-height flex column, vertically centered. Portrait only (CONTEXT.md D-19).

**Background:** `bg-background` (#001233) with two radial blur decorative blobs:
- Top-left: `bg-secondary/10` at 50% viewport width, `blur-[120px]`
- Bottom-right: `bg-primary-container/30` at 50% viewport width, `blur-[120px]`

**Structure (top to bottom):**
1. Logo icon container — 80x80px, `rounded-xl bg-primary-container emerald-glow` — contains placeholder Material Symbol (tracked in icon inventory per CONTEXT.md D-13)
2. App name — "SweatStakes" — Display size (40px / extrabold)
3. Tagline — "Your friends are watching. Are you?" — Body size (16px / regular), `text-on-surface-variant` (Claude's discretion — CONTEXT.md)
4. Glass panel card (`glass-panel` + `rounded-xl border border-outline-variant/15`) containing:
   - Google sign-in button (see button spec below)
   - Inline error message area (hidden by default, visible on failure)

**Google Sign-In Button (CONTEXT.md D-03):**
- Full width, `rounded-full`, `py-5`
- Background: `bg-gradient-to-br from-secondary to-on-tertiary-container` (emerald gradient)
- Text: `text-on-secondary-fixed` (#00210c — near-black, high contrast on emerald)
- Content: Google "G" SVG icon (20x20, original brand colors preserved inside the button) + "Sign in with Google" label
- Font: 16px / 700 bold
- Loading state (CONTEXT.md D-15): spinner replaces icon, button `disabled`, opacity reduced to 60%
- Error state (CONTEXT.md D-09): button re-enables, error message appears below

**Error message (CONTEXT.md D-09):**
- Appears inline below the Google button
- Text: "Sign-in failed. Please try again." — `text-error` (#ffb4ab), 14px, center-aligned
- Icon: `lucide-react` `AlertCircle` at 16px inline — tracked in icon inventory

**No animation** — everything renders immediately, no transitions on load (CONTEXT.md D-04).

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Touch targets | Minimum 44x44px on all interactive elements (iOS HIG) |
| Color contrast | `on-surface` (#d8e2ff) on `background` (#001233) — 12.3:1 (exceeds WCAG AAA) |
| Color contrast (accent) | `on-secondary-fixed` (#00210c) on `secondary` (#66dd8b) — 8.1:1 (exceeds WCAG AAA) |
| Color contrast (error) | `error` (#ffb4ab) on `background` (#001233) — 5.2:1 (WCAG AA) |
| Focus visible | shadcn/ui default focus rings, overridden to `ring-secondary/50` |
| Disabled states | `opacity-60` + `cursor-not-allowed` on disabled buttons |
| ARIA | Sign-in button: `aria-busy="true"` during loading; error message: `role="alert"` |
| Safe areas | `env(safe-area-inset-*)` applied to bottom nav and root layout |
