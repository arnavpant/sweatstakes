  <action>
**Step 1 — Install Vercel CLI (if not already installed):**

```bash
npm install -g vercel
```

**Step 2 — Deploy to Vercel:**

Run `vercel` in the project root. This will:
1. Prompt to link to a Vercel account (or use existing login)
2. Auto-detect Next.js framework
3. Deploy to a preview URL

If `vercel` CLI prompts for project setup:
- Set up and deploy: Yes
- Which scope: (user's account)
- Link to existing project: No (create new)
- Project name: sweatstakes (or accept default)
- Directory: ./
- Override settings: No

**Step 3 — Set environment variables in Vercel:**

```bash
vercel env add NEXT_PUBLIC_SUPABASE_URL production
vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production
```

When prompted, enter the same values from the local `.env.local` file.

Also set for preview and development environments:
```bash
vercel env add NEXT_PUBLIC_SUPABASE_URL preview
vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY preview
```

**Step 4 — Deploy to production:**

```bash
vercel --prod
```

This deploys to the production URL (e.g., sweatstakes.vercel.app).

**Step 5 — Record the deployed URL:**

Note the production URL from the output. It will be needed for the next step (OAuth redirect URI configuration).

**Step 6 — Update `.env.local.example` to document the Vercel deployment note:**

Add a comment section to `.env.local.example`:

```
# Supabase — get from Supabase Dashboard > Project Settings > API
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here

# Deployment notes:
# After deploying to Vercel, add the following to Supabase Auth redirect URIs:
#   https://<your-project>.vercel.app/auth/callback
# Also update Google Cloud Console OAuth redirect URIs to include:
#   https://<supabase-project-ref>.supabase.co/auth/v1/callback
# (This should already be set from Plan 01 setup, but verify it covers production use)
```

**Step 7 — Verify the deployment loads:**

```bash
curl -s -o /dev/null -w "%{http_code}" https://<project-url>.vercel.app
```

Should return 200 or 307 (redirect to /login).
