    **3. Create `src/components/check-in/camera-view.tsx`** -- `'use client'`

    The main camera orchestrator implementing the state machine from 03-RESEARCH.md Pattern 1.

    State machine phases (use useState, NOT Zustand -- keep it local to this component tree):
    ```
    type CameraPhase = 'rear-capture' | 'selfie-countdown' | 'selfie-capture' | 'preview' | 'submitting' | 'done'
    ```

    State: `phase`, `rearBlob: Blob | null`, `selfieBlob: Blob | null`, `compositeBlob: Blob | null`, `countdown: number`, `error: string | null`.

    Implementation flow:
    a. On mount (`useEffect`), open rear camera via `navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment', width: { ideal: 1920 }, height: { ideal: 1080 } } })`. Attach stream to a `<video ref>` element. Set phase to `'rear-capture'`. Handle errors (no camera, permission denied) by setting error state with a user-friendly message.

    b. **Rear capture phase**: Show the video feed full-screen. Show CaptureButton at bottom center. Show a flip button (icon: `cameraswitch` Material Symbol or Lucide `SwitchCamera`) in the top-right corner -- per D-06, flip button only available during rear camera phase. The flip button toggles between `environment` and `user` facingMode by stopping current tracks and opening a new stream.

    c. When user taps shutter (CaptureButton.onCapture): Capture the current video frame to a canvas -> create a Blob via `canvas.toBlob('image/jpeg', 0.9)`. Store as `rearBlob`. Stop rear camera tracks. Open front camera with `facingMode: 'user'`, `width: { ideal: 640 }`, `height: { ideal: 480 }`. Set phase to `'selfie-countdown'`, countdown to 3.

    d. **Selfie countdown phase**: Show front camera video feed full-screen. Apply `transform: scaleX(-1)` CSS to the video element (mirror effect per Pitfall 3, for preview only). Show CountdownOverlay with current countdown number. Use `setInterval` of 1 second to decrement countdown. When countdown hits 0, capture the front camera frame to canvas (do NOT apply the CSS mirror to canvas capture -- canvas reads raw video). Store as `selfieBlob`. Stop front camera tracks. Set phase to `'selfie-capture'` then immediately to `'preview'`.

    e. **Composite creation**: After both blobs exist, create composite using canvas (03-RESEARCH.md Pattern 3):
       - Create canvas at rearImg dimensions.
       - Draw rearImg full-size.
       - Draw selfie in top-left: 25% of canvas width, 16px margin, 12px border-radius clip, 3px white border.
       - `canvas.toBlob('image/jpeg', 0.92)` -> store as `compositeBlob`.
       - Use `createImageBitmap()` for both blobs to create drawable images.

    f. **Preview phase**: Render by passing `compositeBlob` to the PhotoPreview component.

    g. **Cleanup**: `useEffect` return function must stop all active tracks: `stream.getTracks().forEach(track => track.stop())`. Also listen for page visibility change to stop tracks when user navigates away.

    h. **Error handling**: If `navigator.mediaDevices` is undefined (HTTP context), show "Camera requires a secure connection (HTTPS)". If permission denied, show "Camera permission denied. Please allow camera access in your browser settings." Use the error color from design tokens.

    Full-screen layout: The component should fill the entire viewport. Use `fixed inset-0 z-50 bg-background` to overlay everything including the bottom nav. Add a close/back button (X icon or left arrow) in top-left to navigate back to dashboard via `router.back()`.

    a. Set submitting state to true. Disable both buttons. Show Loader2 spinner on Submit button.
    b. Compress: `import imageCompression from 'browser-image-compression'`. Create a File from the Blob: `new File([compositeBlob], 'check-in.jpg', { type: 'image/jpeg' })`. Compress with options: `{ maxSizeMB: 0.5, maxWidthOrHeight: 1920, useWebWorker: true, fileType: 'image/jpeg' }`.

Upload: `supabase.storage.from('check-in-photos').upload(fileName, compressed, { cacheControl: '31536000', contentType: 'image/jpeg', upsert: false })`. Handle error with user-friendly message.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Supabase Storage | Untrusted file blob uploaded directly from browser to storage bucket |
| Browser -> Server Action | photoUrl and checkedInDate from client passed to submitCheckInAction |
| Camera API -> Browser | Media stream data from device camera into browser context |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Denial of Service | photo-preview.tsx upload | mitigate | Compress to maxSizeMB: 0.5 via browser-image-compression before upload; prevents storage exhaustion |
| T-03-08 | Tampering | photo-preview.tsx file type | mitigate | Force contentType: 'image/jpeg' on upload; Supabase Storage validates content type |
| T-03-09 | Spoofing | Supabase Storage upload | mitigate | RLS policy restricts INSERT to authenticated users only; bucket-level access control |
| T-03-10 | Information Disclosure | camera-view.tsx error messages | mitigate | Show generic camera errors; never expose device IDs or raw API errors to UI |
| T-03-11 | Repudiation | photo-preview.tsx | accept | No deletion allowed (D-18); check-ins are permanent once submitted |
</threat_model>
